# Курсовая работа по алгоритмам и структурам данных

Информационная система для обслуживания читателей в библиотеке.

## Описание

Проект представляет собой информационную систему для управления библиотекой, которая осуществляет ввод, хранение, обработку и вывод данных о:

- **Читателях библиотеки** (ФИО, билет, год рождения, адрес, место работы)
- **Книгах в фонде библиотеки** (Шифр, авторы, название, издательство, год, экземпляры)
- **Выдаче и приеме книг** (Связь читателя и книги с датами выдачи и возврата)

## Технологии

- **Язык программирования:** Kotlin 2.1.0
- **Платформы:** Desktop (JVM), Web (WASM) через Kotlin Multiplatform
- **UI:** Compose Multiplatform (Material 3)
- **Архитектура:** MVVM (Model-View-ViewModel)
- **Система сборки:** Gradle (Kotlin DSL)

## Реализованные алгоритмы и структуры (Вариант 624)

Согласно заданию (вариант 624), реализованы следующие компоненты:

1.  **Хеш-таблица с открытым хешированием** — хранение читателей, первичный ключ: номер билета.
2.  **АВЛ-дерево поиска** — хранение книг, упорядоченных по шифру.
3.  **Слоеный список (Skip List)** — учет выдачи книг, упорядоченный по шифру книги.
4.  **Сортировка включением (Insertion Sort)** — для поддержания порядка в списках.
5.  **Симметричный обход дерева (In-order traversal)** — для вывода данных и поиска.
6.  **Алгоритм Бойера-Мура** — для регистронезависимого поиска подстроки в ФИО читателей, названиях и авторах книг.

## Основные возможности

### Читатели
- Регистрация и удаление читателей.
- Поиск по номеру билета (O(1) в среднем) и по ФИО (Boyer-Moore).
- **Валидация:** Нельзя удалить читателя, у которого на руках остались книги.

### Книги
- Учет книжного фонда с автоматическим расчетом доступных экземпляров.
- Поиск по шифру и фрагментам названия/автора (Boyer-Moore).
- Визуализация карточек книг в стиле реальных книжных обложек.
- **Валидация:** Нельзя удалить книгу, если хотя бы один экземпляр находится у читателя.

### Выдача и возврат
- Упрощенный процесс выдачи через **Autocomplete поля** (умный поиск по читателям и книгам).
- Автоматическая проверка наличия свободных экземпляров перед выдачей.
- Учет истории возвратов.

### Настройки
- Генерация тестовых данных (читатели, книги разных категорий).
- Возможность частичной или полной очистки базы данных.

## Сборка и запуск

### Требования
- JDK 17 или выше
- Gradle 8.5+

### Команды

```bash
# Запуск десктопной версии (JVM)
./gradlew :app:run

# Сборка WASM-версии (Web)
./gradlew :app:wasmJsBrowserDistribution

# Запуск тестов
./gradlew test
```
