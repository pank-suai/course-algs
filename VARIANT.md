## Предметная область "Обслуживание читателей в библиотеке"


### Общие требования

Информационная система для предметной области «Обслуживание читателей в библиотеке» должна осуществлять ввод, хранение, обработку и вывод данных о:

* читателях;
* книгах;
* выдаче и приеме книг от читателей.

### Данные о читателях

Данные о каждом читателе должны содержать:

* **Номер читательского билета** — строка формата `ANNNN-YY`, где:
* `A` — буква, обозначающая права доступа:
* **А** — только абонемент;
* **Ч** — только читальный зал;
* **В** — читальный зал и абонемент.


* `NNNN` — порядковый номер регистрации (цифры);
* `YY` — последние две цифры года регистрации.


* **ФИО** — строка.
* **Год рождения** — целое число.
* **Адрес** — строка.
* **Место работы/учебы** — строка.

> **Примечание:** Длина строк (кроме номера билета) определяется студентом самостоятельно.

**Организация данных:** Информация о читателях должна быть организована в виде **хеш-таблицы**, где первичным ключом является «номер читательского билета». Метод хеширования определяется вариантом задания.

---

### Данные о книгах

Данные о каждой книге должны содержать:

* **Шифр** — строка формата `NNN.MMM`, где:
* `NNN` — номер тематического раздела;
* `MMM` — порядковый номер книги в разделе.


* **Автор(ы)** — строка.
* **Название** — строка.
* **Издательство** — строка.
* **Год издания** — целое число.
* **Количество экземпляров всего** — целое число.
* **Количество экземпляров в наличии** — целое число.

**Организация данных:** Информация о книгах должна быть организована в виде **АВЛ-дерева поиска**, упорядоченного по «Шифру».

---

### Учет выдачи и приема книг

Данные должны содержать:

* **Номер читательского билета** (соответствует формату в данных о читателях).
* **Шифр книги** (соответствует формату в данных о книгах).
* **Дата выдачи** — строка.
* **Дата возврата** — строка.

**Организация данных:** Данные должны быть организованы в виде **списка**, упорядоченного по первичному ключу — «Шифр». Вид списка и метод сортировки определяются вариантом задания.

> **Важные замечания:**
> 1. Наличие записи  означает выдачу книги  читателю . Отсутствие записи — книга не выдавалась.
> 2. Связь «многие-ко-многим»: один читатель может взять много книг, одна книга может быть у разных читателей.
> 
>

### Функциональные возможности системы

Система должна обеспечивать выполнение следующих операций:

#### Работа с читателями:

* Регистрация нового читателя.
* Снятие с обслуживания (удаление).
* Просмотр всех зарегистрированных читателей.
* Очистка всех данных о читателях.
* **Поиск по номеру билета:** вывод всех сведений о читателе и шифров книг, которые у него на руках.
* **Поиск по ФИО:** вывод списка найденных читателей (билет + ФИО).

#### Работа с книгами:

* Добавление новой книги.
* Удаление сведений о книге.
* Просмотр всех имеющихся книг.
* Очистка данных о книгах.
* **Поиск по шифру:** вывод всех сведений о книге, а также ФИО и номеров билетов читателей, у которых она находится.
* **Поиск по фрагментам автора/названия:** вывод списка книг (шифр, автор, название, изд-во, год, наличие).

#### Процесс обслуживания:

* Регистрация выдачи экземпляра (только если «в наличии» ).
* Регистрация приема экземпляра.

---

### 9.1.6. Технические особенности реализации

1. **Просмотр данных:** Состав полей при общем просмотре определяется студентом (минимум два поля).
2. **Поиск по ФИО:** Метод поиска выбирается студентом и сравнивается с альтернативами.
3. **Поиск по фрагментам:** Осуществляется путем систематического обхода АВЛ-дерева. Для поиска подстроки в строке используется алгоритм, указанный в варианте.
4. **Корректность данных:** При выдаче/приеме книг поле «Количество экземпляров в наличии» должно автоматически обновляться.
5. **Целостность данных:** При удалении читателя или книги необходимо обработать ситуации, когда за читателем числятся книги или книга находится на руках.
